    VS Code 开发 Hello JSP 程序（无 Java Servlet 插件，使用 Community Server Connectors）(验证未通过)

    本文全程不依赖 Java Servlet 插件，仅通过核心必备插件+规范配置，实现 JSP 项目的开发、部署与运行，步骤简洁且可落地。

    一、前置准备

    1. 软件环境
      • 已安装 VS Code（最新稳定版）
      • 已安装 OpenJDK 23+（配置 JAVA_HOME 环境变量并添加到系统 Path，CMD 输入 java -version 和 javac -version 验证生效）
      • 已下载 Tomcat 11+（解压至非中文、无空格目录，例如 D:\apache-tomcat-11，双击 bin/startup.bat（Windows）/ startup.sh（Mac/Linux）可正常启动 Tomcat 首页 http://localhost:8080）
    2. 环境验证
      • Tomcat 启动后，浏览器访问 http://localhost:8080 显示 Tomcat 欢迎页，说明服务器正常；后续需先双击 bin/shutdown.bat 停止 Tomcat，再进行项目配置

    二、步骤1：安装必备插件

    打开 VS Code，按下 Ctrl+Shift+X 打开扩展市场，仅需安装以下 2 款核心插件，满足 JSP 开发基础需求： 1. Extension Pack for Java - 核心功能：提供 Java 语法编译、代码提示、类导入支持，是 JSP 代码运行的底层依赖，解决 Java 代码报红问题 - 注意：安装后会自动携带相关依赖插件，无需额外配置 2. Community Server Connectors - 核心功能：替代 Tomcat for Java，实现 Tomcat 服务器的添加、启停、JSP 项目部署，无需手动复制文件到 Tomcat 目录 - 关键：插件安装完成后，必须重启 VS Code 使其生效，否则无法调出 Servers 面板

    三、步骤2：手动创建 hello_jsp 标准项目目录结构

    VS Code 无内置 JSP 模板，需手动创建规范目录（确保 Community Server Connectors 和 Tomcat 能识别，无关插件）： 1. 新建文件夹 hello_jsp（作为项目根目录），通过 VS Code 的 File -> Open Folder 打开该文件夹 2. 在项目根目录下，严格按照以下结构创建目录和文件（不可随意改名，核心是 webapp 目录）： 

    hello_jsp
├── webapp  // JSP 核心存放目录（Tomcat 默认识别该目录名，必须存在）
│   └── index.jsp  // 待编写的 JSP 首页文件（后续创建）
└── WEB-INF  // 受保护目录（可选，新手创建空目录即可，后续扩展 Servlet 可用）
    ├── classes  // 编译后的 Java class 文件目录（空目录）
    └── lib      // 第三方 Jar 包存放目录（空目录）
    3. 验证：左侧资源管理器中显示上述目录结构，说明目录创建完成

    四、步骤3：编写 Hello JSP 核心代码（无插件语法支持也可运行）

    即使无 Java Servlet 插件，JSP 代码仍可正常运行，仅需保证代码规范和编码正确： 1. 在 webapp 目录下，右键 -> 「新建文件」，命名为 index.jsp 2. 打开 index.jsp，粘贴以下完整代码（含中文乱码处理、Java 代码片段，可直接运行）： 

    <%-- 页面核心配置：指定 UTF-8 编码防止中文乱码，声明语言为 Java --%>
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%-- 手动导入 Date 类，解决 Java 类报红问题（依赖 Extension Pack for Java） --%>
<%@ page import="java.util.Date" %>
<html>
<head>
    <title>Hello JSP (No Java Servlet Plugin)</title>
</head>
<body>
    <!-- 核心显示内容：居中蓝色大号字体，无需插件支持，浏览器可直接解析 -->
    <h1 style="color: #2f5496; text-align: center;">Hello JSP!</h1>
    <%-- JSP 脚本片段：执行 Java 代码，输出当前系统时间 --%>
    <%
        // 创建 Date 对象获取当前时间
        Date currentTime = new Date();
        // 向页面输出时间内容
        out.println("<p style='text-align: center;'>当前系统时间：" + currentTime + "</p>");
    %>
</body>
</html>
    3. 保存文件：按下 Ctrl+S 保存，即使 VS Code 未显示 JSP 语法高亮（无插件导致），也不影响项目部署和运行

    五、步骤4：配置 Community Server Connectors + Tomcat（核心步骤）

    1. 打开 Servers 面板

    • 方式1：点击 VS Code 左侧边栏的「Servers」图标（安装 Community Server Connectors 后新增，形似服务器图标）
    • 方式2：按下 Ctrl+Shift+P 打开命令面板，输入 Servers: Show Servers View，回车即可调出面板

    2. 添加 Tomcat 服务器

    1. 在 Servers 面板中，点击「Create New Server...」

    2. 在弹出的选项中选择No, user server on disk

    3. 然后选中Tomcat安装目录，如apache-tomcat-11

    4. 验证：Servers 面板中显示该 Tomcat 服务器，状态为「Stopped」（未启动），说明添加成功

    3. 部署 hello_jsp 项目到 Tomcat

    1. 在 Servers 面板中，右键点击已添加的 Tomcat 服务器，选择「Add Deployment」（添加部署项）
    2. 在部署类型中，选择「Folder」（因项目是手动创建的目录结构，非 war 打包文件）
    3. 点击「Browse」，必须选择 hello_jsp 项目下的 webapp 目录（Tomcat 仅识别该目录下的 JSP 文件）
    4. 「Deployment Name」：填写 hello_jsp（自定义，便于识别，与访问路径直接相关）
    5. 点击「Finish」，完成项目部署（Tomcat 服务器下会显示 hello_jsp 部署项，说明部署成功）

    六、步骤5：启动服务器并验证 Hello JSP 结果

    1. 启动 Tomcat 服务器

    1. 方式1：在 Servers 面板中，右键点击 Tomcat 服务器，选择「Start Server」（启动服务器）
    2. 方式2：选中 Tomcat 服务器，点击 Servers 面板上方的「绿色三角」启动图标
    3. 启动验证：VS Code 终端会输出 Tomcat 启动日志，当出现 Server startup in XXX ms 字样时，说明 Tomcat 启动成功

    2. 访问并验证项目

    1. 手动访问：打开浏览器，在地址栏输入 http://localhost:8080/hello_jsp
      • 路径说明：8080 是 Tomcat 默认端口，hello_jsp 是步骤4.3中填写的「Deployment Name」
    2. 快捷访问：在 Servers 面板中，右键点击 hello_jsp 部署项，选择「Open in Browser」，自动打开浏览器访问
    3. 成功标志：
      • 浏览器页面居中显示蓝色大号字体 Hello JSP!
      • 下方显示当前系统时间，无中文乱码，说明 JSP 代码执行正常（即使 VS Code 无语法高亮，浏览器仍可正常解析）

    七、项目修改与重部署（便捷操作）

    1. 修改代码：直接编辑 index.jsp 文件，修改后按下 Ctrl+S 保存
    2. 重部署项目：在 Servers 面板中，右键点击 hello_jsp 部署项，选择「Redeploy」（快速重部署，无需重启 Tomcat）
    3. 生效验证：刷新浏览器，即可看到修改后的效果（JSP 支持热更新，重部署后立即生效）
    4. 停止服务器：右键点击 Tomcat 服务器，选择「Stop Server」（或点击 Servers 面板上方的「红色方形」停止图标）

    八、常见问题排查（无 Java Servlet 插件场景）

    1. JSP 文件无语法高亮
      • 解决方案：这是无 Java Servlet 插件的正常现象，不影响运行；若需高亮，可右键 index.jsp -> 「选择语言模式」-> 手动选择「JSP」，临时开启语法高亮
    2. 访问 404 错误
      • 解决方案：1. 确认部署时选择的是 webapp 目录；2. 确认访问路径中的 hello_jsp 与「Deployment Name」一致；3. 确认 Tomcat 已正常启动；4. 确认 index.jsp 在 webapp 目录下（非 WEB-INF 目录）
    3. 中文乱码
      • 解决方案：1. 确保 JSP 页面 <%@ page %> 指令中 charset=UTF-8；2. 打开 Tomcat 解压目录 conf/server.xml，在 Connector 标签中添加 URIEncoding="UTF-8"，配置如下： 
        <Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           URIEncoding="UTF-8"/>
    4. Java 代码报红（如 Date 类）
      • 解决方案：1. 确认 Extension Pack for Java 已安装并启用；2. 手动添加 <%@ page import="java.util.Date" %> 导入语句；3. 确认 JDK 配置正常（JAVA_HOME 路径无中文/空格）
    5. 端口被占用
      • 解决方案：1. 打开 Tomcat conf/server.xml，修改 Connector 标签的 port 属性（如 8080 改为 8081）；2. 重启 Tomcat 服务器即可

    九、总结

    1. 核心原则：无 Java Servlet 插件完全可实现 JSP 基础开发，插件仅提供语法高亮/补全，不影响代码运行
    2. 关键配置：Extension Pack for Java（底层依赖）+ Community Server Connectors（服务器管理）+ 规范 webapp 目录结构
    3. 访问路径：http://localhost:8080/hello_jsp（hello_jsp 为部署名称），成功标志是页面显示 Hello JSP! 和当前系统时间
    4. 便捷操作：支持「Redeploy」快速更新代码，无需手动复制文件到 Tomcat 的 webapps 目录，提升开发效率
